# Docs Hardening Template

Use this when the task is improving documentation clarity, onboarding experience, or README
quality. Defines the target audience and specific weaknesses to address.

```md
# Documentation Session

## Objective
[What documentation needs improvement?]

## Audience
- new developer
- maintainer
- user
- recruiter
- interviewer
- contributor

## Problem With Current Docs
[What is weak, missing, confusing, or outdated?]

## Required Improvements
- setup clarity
- usage examples
- architecture explanation
- contribution instructions
- troubleshooting
- project purpose
- credibility / proof of execution

## Constraints
- be specific, not generic
- avoid inflated claims
- reflect actual repo behavior
- keep examples accurate

## Desired Output
- revised README sections
- new docs page
- setup guide
- FAQ / troubleshooting
```
