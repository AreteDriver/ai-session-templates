# Repo Review Template

Use this when asking the agent to analyze a repo for interview prep, OSS polish, product
positioning, or credibility audit. Evaluates first impressions and actionable fixes.

```md
# Repo Review Session

## Repo
[name + link/path]

## Purpose
[What the repo is supposed to be]

## Review Goal
- interview narrative
- OSS polish
- recruiter credibility
- product readiness
- monetization readiness
- architecture review

## Evaluate These Areas
- first impression
- README clarity
- install/run experience
- evidence of real execution
- architecture coherence
- release/CI hygiene
- trustworthiness
- novelty
- weakest points

## Required Output
1. What this repo signals to a stranger in 30 seconds
2. Strengths
3. Weaknesses
4. Top fixes by impact
5. Suggested positioning language
```
